플레이어 캐릭터를 키보드로 움직이기 구현 및 FPS 확인



&nbsp;AI 대화 기록



Q1: Pygame에서 키보드 입력을 받아 원을 움직이려면 어떻게 해야 할까?



\- AI 답변: `pygame.key.get\_pressed()` 사용, 눌린 키에 따라 `x`와 `y` 좌표를 변경  

\- 시행착오: 처음엔 이벤트만 사용하려고 했지만, 계속 누르면 반응이 부드럽지 않았음  

\- 최종 적용 결과: 방향키를 누르면 원이 부드럽게 5픽셀씩 이동 성공



---



Q2: 원이 화면 밖으로 나가는 것을 방지하려면 어떻게 해야 할까?



\- AI 답변: 좌표 이동 전에 화면 경계를 체크 (`0 <= x <= 화면폭`, `0 <= y <= 화면높이`)  

\- 시행착오: 처음에는 조건문 위치가 잘못되어 원이 화면 밖으로 나가기도 함  

\- 최종 적용 결과: 원이 화면 안에서만 움직이도록 경계 처리 완료



---



Q3: FPS를 화면에 표시하면서 원과 겹치지 않게 하려면?



\- AI 답변: `pygame.font.SysFont`로 텍스트 생성, `screen.blit()`로 좌측 상단 표시  

\- 시행착오: 처음에 FPS 텍스트를 그리기 전에 원을 먼저 그려서 겹치거나 지워짐  

\- 최종 적용 결과: `screen.fill()` → `원 그리기` → `FPS 표시` 순서로 처리하여 겹치지 않음
